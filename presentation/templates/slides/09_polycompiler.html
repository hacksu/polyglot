polycompiler — Python + JavaScript

A library by evanzhoudev that merges separate Python and JS files into one file runnable by both CPython and Node.js.  github.com/evanzhoudev/polycompiler

Install + use

npm install -g polycompiler

# Merge Python + JS into one file
polycompiler script.py script.js -o both.py

# Same file, two runtimes:
python both.py
node   both.py

Why Python+JS is harder

Python and JavaScript share similar surface syntax — you can't just hide one inside the other's comment blocks. polycompiler uses semantic divergence: expressions that evaluate differently in each runtime to branch execution.

script.py script.js │ │ └──── polycompiler ──┘ │ both.py ┌────┴────┐ │ │ python3 node runs it runs it

How it branches

  • Detects runtime via expressions that differ
  • (1).__class__.__name__ → 'int' in Python, error in JS
  • typeof undefined → valid in JS, NameError in Python
  • Uses these divergences to route to the right code path
