04 — Binary Polyglots

Binary Polyglots — Images

File format parsers have their own ignored regions. A single file can be a valid JPEG and a valid ZIP archive simultaneously.

┌──────────────────────────────────────┐ │ polyglot.jpg (on disk) │ ├──────────────────────────────────────┤ │ FF D8 FF E0 ... ← JPEG magic bytes │ │ [JPEG image data ................] │ │ [JPEG image data ................] │ │ FF D9 ← JPEG end marker │ ├──────────────────────────────────────┤ │ PK\x03\x04 ... ← ZIP local header │ │ [ZIP file entries ................] │ │ PK\x05\x06 ... ← end-of-central- │ │ directory │ └──────────────────────────────────────┘
Image viewer: reads from the start → sees valid JPEG ✓
ZIP tool:    reads from the end  → sees valid ZIP ✓
Neither parser notices the other's data.

Why it works

  • JPEG spec: parse forward from magic bytes
  • ZIP spec: find end-of-central-dir from the back
  • JPEG ignores all data after FF D9
  • ZIP ignores bytes before its local headers

Real-world examples

  • CTF forensics — classic puzzle type
  • GIFAR — GIF+JAR (Java applet in image)
  • PDF+ZIP — common in competitions
  • Malware — evade file-type content filters
Try it: cat image.jpg archive.zip > polyglot.jpg
Open in an image viewer, then run unzip polyglot.jpg
