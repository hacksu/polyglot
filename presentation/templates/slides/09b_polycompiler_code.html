How the Two Lines Work

1 // (lambda: exec("print('Hello Python!')", globals()) or 1)()
lambda: eval("console.log('Hello JS!')")
PYTHON READS IT
1
//    ← floor division operator
(lambda: exec("...", globals()) or 1)()
    ↑ lambda is called immediately
     exec() runs the Python code string
     "or 1" ensures result is truthy
     1 // result → integer, discarded
lambda: eval("...")
    ↑ lambda created but never called
     eval() never runs → JS code ignored
JAVASCRIPT READS IT
1
//    ← LINE COMMENT starts here
(lambda: exec("...", globals()) or 1)()
    ↑ entire rest of line 1 is ignored
     Python code never seen by JS
lambda:
     ↑ LABELED STATEMENT (rare JS syntax)
     "lambda" is just a label name
eval("console.log('Hello JS!')")
    ↑ this runs! JS code executes
